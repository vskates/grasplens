GraspLens

A runtime assurance benchmark for learned robotic grasping. It tests whether formal geometry checks and sparse activation probes can reject unsafe grasp choices under distribution shift.

Problem

The learned scorer fails where the test distribution changes.

The checked-in validation artifact is an offline generated-scene benchmark with exact geometry labels. The learned scorer is trained on a shortcut-biased imitation target: in train scenes, red objects usually indicate the target. In shifted test scenes, red distractors are common.

This gives a controlled setting for studying grasp ranking under distribution shift: every candidate has a learned score, hidden activations, and formal predicate labels. The runtime layer can then be measured against known unsafe selections.

Experimental envelope

The benchmark is specified in metric lab coordinates.

Robot envelope 7-DoF arm, 855 mm reach, 3 kg payload

Franka Research 3-class tabletop arm with a 2F-85-class parallel-jaw gripper and 85 mm opening.

Scene map 0.72 m x 0.72 m top-down ROI

Shelf interior is 0.662 m x 0.662 m; object boxes are 54-94 mm by 40-83 mm.

Candidate geometry 115.2 mm x 39.6 mm grasp rectangle

The approach sweep samples 18.0-129.6 mm along the approach vector using seven offsets.

Predicate gates 8.6 mm minimum clearance threshold

Collision, wrong-object contact, semantic hazard, target contact, and clearance are computed per candidate.

Validation boundary

Current metrics are from the checked-in offline benchmark: 2,000 train scenes, 1,000 shifted test scenes, and 50,210 shifted test candidates. The repository defines the Isaac/ROS2 candidate interface, but does not include robot or Isaac rollout logs.

System model

Where the learned model sits in the assurance pipeline.

Input scene x, candidates G(x)

Each candidate has pose, object-relative features, color/semantic tags, and local clearance features.

Learned model MLP-style grasp scorer

The scorer outputs utility score s_theta(x,g) and hidden activation z_theta(x,g).

Safety layer geometry + probes + conformal gate

Hard predicates define S_geo; sparse probes estimate latent unsafe risk; conformal calibration produces C_alpha.

Method

One learned scorer, four selection rules.

1 Generate candidates

Top-down shelf scene, object boxes, obstacles, grasp rectangles.

2 Score grasps

MLP-style scorer ranks candidates and exposes hidden activations.

3 Check geometry

Hard predicates reject collision, wrong-object, low-clearance and hazard cases.

4 Probe latent risk

Sparse logistic probes estimate unsafe concepts from hidden activations.

5 Calibrate

Split conformal prediction gates aggregate unsafe risk.

6 Select grasp

Choose the highest learned score among accepted candidates.

Formal setup

The selector separates utility from acceptability.

For scene x and candidate set G(x), define:

S_geo(x) = { g in G(x) : h_j(x, g) >= 0 for every hard predicate j }

The full selector uses learned score only after hard filtering:

g*(x) = argmax score_theta(x, g)
        subject to g in S_geo(x)
        and unsafe not in C_alpha(z_theta(x, g))

The conformal set is calibration for latent residual risk. It does not replace the geometric predicates.

Benchmark

Generated scenes with exact labels.

Each scene contains one target, several distractors, shelf boundaries, random obstacles, and fragile or hazardous tags. Candidate grasps are oriented rectangles sampled around each object.

Labels are computed directly from geometry: target contact, wrong-object contact, collision, low approach clearance, semantic hazard, and final success. The benchmark is deterministic with respect to the recorded seed and candidate generator settings.

  • Policy backend in this run: NumPy implementation of the MLP-style scorer.
  • Train candidates: 99,560.
  • Shifted test candidates: 50,210.
  • Conformal test coverage: 0.913.

Algorithms

Algorithms implemented in the repository.

Scene generator

Samples products, distractors, obstacles, hazards, and a train/test color shortcut.

Grasp sampler

Builds oriented rectangular candidates around each object with approach vectors.

Geometry predicates

Checks shelf collision, obstacle collision, target contact, non-target contact, and clearance.

Learned scorer

MLP-style scorer trained on noisy shortcut-biased imitation rewards.

Sparse probes

L1 logistic probes map hidden activations to safety concepts and top hidden dimensions.

Runtime selector

Compares max-score baseline, hard geometry, calibrated probe rejection, and the combined filter.

Limitations

Scope of the current run.

Covered in this run

  • A reproducible shortcut failure in learned grasp ranking.
  • A metric reference envelope for tabletop shelf-picking candidates.
  • A hard safety layer that rejects unsafe candidates by construction.
  • Activation-level probes that track safety concepts on shifted scenes.
  • A clean interface for replacing generated candidates with robot-generated candidates.

Outside current run

  • Hardware execution logs or Isaac Sim rollout traces.
  • Generalization to camera noise, depth artifacts, or control errors.
  • Safety for unmodeled hazards outside the predicate set.
  • Interpretability claims for a full VLA model.
